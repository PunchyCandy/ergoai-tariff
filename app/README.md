# ErgoAI Tariff Application

This app now supports two modes:

- the original facts-only Ergo demo in [`main.ergo`](./main.ergo)
- a real Python CLI in [`main.py`](./main.py) that accepts shipment inputs and returns computed duty results
- a retrieval-backed HTS quote flow powered by the local corpus in `rag/data/hts_policy.json`

## What The CLI Does

`main.py` loads the Ergo knowledge base from:

- `data/products.ergo`
- `data/shipments.ergo`
- `rules/decision.ergo`

It then either:

- evaluates an existing sample shipment id
- evaluates a new shipment you provide at runtime
- resolves the shipment HS code from either the existing product catalog or a runtime `--hs-code`
- retrieves HTS policy records from the local HTS corpus and injects them as temporary Ergo facts
- or prompts you interactively in the terminal

## Requirements

You need ErgoAI available through:

- `ERGOROOT`
- `XSBARCHDIR`

Those can be supplied through your shell environment or a `.env` file.

## Commands

List the sample shipments already in the knowledge base:

```bash
python3 app/main.py list
```

Evaluate a sample shipment:

```bash
python3 app/main.py sample s1
```

Evaluate a runtime shipment using an existing product id:

```bash
python3 app/main.py quote \
  --origin china \
  --import-country usa \
  --product-id p_router \
  --value 1000 \
  --date 2025-11-10
```

Evaluate a runtime shipment with a custom HS code:

```bash
python3 app/main.py quote \
  --origin china \
  --import-country usa \
  --product-id custom_router \
  --hs-code 8517.62.0000 \
  --category electronics \
  --description wireless_router \
  --value 1000 \
  --date 2025-11-10
```

Evaluate a shipment using a code that exists only in the retrieval corpus:

```bash
python3 app/main.py quote \
  --origin taiwan \
  --import-country usa \
  --product-id custom_chip \
  --hs-code 8542.31.0000 \
  --category electronics \
  --description integrated_circuit \
  --value 2500 \
  --date 2025-11-10 \
  --json
```

Run in interactive mode:

```bash
python3 app/main.py interactive
```

Return machine-readable JSON only:

```bash
python3 app/main.py quote \
  --origin japan \
  --import-country usa \
  --product-id p_parts \
  --value 1200 \
  --date 2026-02-02 \
  --json
```

## Output Shape

The CLI returns structured output with:

- `input`: the shipment facts used for reasoning
- `product`: the resolved product metadata
- `retrieved_policy`: the HTS policy records pulled from the local retrieval corpus for the resolved HS code
- `result`: the computed duty, rate, and Ergo explanation

## Notes

- `app/tmp_combined_rules.ergo` is generated by `main.py` before querying Ergo.
- Duty evaluation now depends on retrieved `tariff_rate` records instead of the static `policy/usa_base.ergo` file.
- Static override and exemption policy files are not part of the active runtime path.
- Retrieval uses exact HTS code matching against `rag/data/hts_policy.json`.
- HTS codes are normalized to the repo's six-digit style such as `8517.62.0000 -> hs8517_62`.
- The facts-only demo remains useful if you want a guaranteed-simple Ergo load path.
